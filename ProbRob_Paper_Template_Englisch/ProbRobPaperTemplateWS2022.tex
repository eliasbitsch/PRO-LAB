\section{Introduction}

This is an altered version of the IEEE conference paper template with simplified formatting and Bibtex bibliography. The \textit{refs.bib} file contains all bibliography sources in Bibtex format. The sources can be cited using the \textit{cite} command \cite{Bishop2006Pattern} (see this document's LaTeX source code). The corresponding entry will be generated automatically in the bibliography.

\vspace{\baselineskip}
Remarks about the paper's structure are marked \color{structure}\textbf{blue}\color{black}, while the template has been filled with blindtext marked in \color{blindtext}\textbf{light gray}\color{black} to roughly indicate each chapter's desired length.

\vspace{\baselineskip}
\color{structure}Your paper's main part (first page until (and including) references) has to \textbf{exactly fill three pages}, similarly to how it's handled at scientific conferences. In order to achieve this, you can fine-tune text length using paragraphs and image/table scaling.\color{black}

\vspace{\baselineskip}
\textbf{Please consider the following aspects when writing your paper:}
\begin{itemize}
    \item \textbf{Structure}: Adhere to IMRAD and include all required chapters. Make sure that potential readers are able to follow one continuous thread (\textit{$=$ roter Faden}) spanning from introduction to summary and outlook.
    
    \item[]
    \item \textbf{Research}: Read and include relevant sources in introduction and state of the art and use these two chapters to introduce the topic, (scientifically) justify what you are doing and prepare readers for your methods.\\
    \textbf{Every claim you make must be backed up by either literature or your presented results.}
    
    \item[]
    \item \textbf{Contribution}: Define a clear contribution your paper provides to tell the reader the main scientific benefit of your paper. Justify your contribution based on the problem you are trying to solve. This problem should in turn arise from the research presented in the state of the art.
    
    \item[]
    \item \textbf{Wording}: Make sure that you use appropriate wording (see established conference papers for inspiration). Explain concepts as clearly as possible without over-complicating explanations or including redundant information. Similarly to your code, use the KISS principle (Keep it Stupid Simple)!
\end{itemize}

\subsection{Figures}
Figures as well as tables have to be referred to in the text. Please use Fig.~\ref{fig:technikumLogo} to refer to figures and Tab.~\ref{tab:leereTabelle} to refer to tables.

\vspace{\baselineskip}
\textbf{Hint for figures in LaTeX:} Embed images as \textit{.pdf} files. PDF files are stored in vector format and can easily be scaled by LaTeX. Furthermore, text stays natively embedded in them. I suggest finding a workflow with a vector-based editor such as DrawIO or Inkscape (links as comments are in this document's source code) to draw figures and natively export them to \textit{.pdf}.

\vspace{\baselineskip}

If you are using information in a figure that is not your own, you must include a reference in the figure's description. There are thee ways to do this (please note that you will need to use \verb!\protect\cite{}! when adding a citation to a figure's description in LaTeX):

\begin{itemize}
    \item[]
    \item[] \textit{(without modification from \cite{Bishop2006Pattern})} 
    \item[$\to$] Image has been directly copied from source.
    \item[]
    \item[] \textit{(with modification from \cite{Bishop2006Pattern})}
    \item[$\to$] Portions of the original source's image have been copied, but parts have been added/modified/omitted by you.
    \item[]
    \item[] \textit{(drawn based on \cite{Bishop2006Pattern})}
    \item[$\to$] You have completely redrawn the image based on the cited source.
    \item[]
\end{itemize}

Directly using other author's figures in publicised work (that includes the master thesis) requires written permission to do so from author and, if present, editor.
Permission is required as soon as a single pixel is adopted from the original image.

\textbf{Therefore, drawing your own image based on the original image (option 3) is generally the best option.}
